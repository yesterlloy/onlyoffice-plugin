## 1. 项目概述

### 1.1 项目范围

制式报告模板编辑器，采用三端分离架构：

| 端 | 技术栈 | 说明 |
|---|--------|------|
| 产品前端 | React + Ant Design + TypeScript | 全部 UI 交互 |
| 薄插件 | OnlyOffice Plugin SDK | 文档操作代理 |
| 产品后端 | Java + Spring Boot + MySQL | 数据与服务 |

### 1.2 核心功能

- 指标库管理（分类、搜索、拖拽）
- 模板编辑（可视化标签、参数配置）
- AI 内容生成（提示词模板 + 本地 LLM）
- 模板保存与预览
- 报告生成

## 4. 依赖与风险

### 4.1 外部依赖

| 依赖项 | 说明 | 风险等级 |
|--------|------|----------|
| MySQL 8.0 | 数据库 | 低 |
| OnlyOffice | 文档编辑器 | 中（版本兼容性） |
| 阿里云 OSS | 文件存储 | 低 |
| 本地 LLM 服务 | AI 内容生成 | 中（稳定性） |

### 4.2 风险识别

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| OnlyOffice API 变更 | 插件功能异常 | 锁定 OnlyOffice 版本，充分测试 |
| 本地 LLM 不稳定 | AI 生成失败 | 增加重试机制，支持多模型切换 |
| 前后端并行开发 | 接口不一致 | 提前定义 API 契约，使用 Mock 数据 |
| 拖拽功能复杂度 | 开发延期 | 使用成熟库（dnd-kit），预留缓冲时间 |

## 6. 验收标准

### 6.1 功能验收

| 功能 | 验收标准 |
|------|----------|
| 指标库加载 | 分类树正确显示，搜索过滤有效 |
| 拖拽插入 | 指标可拖拽到文档，Content Control 正确创建 |
| 参数配置 | 配置面板正确渲染，参数可保存更新 |
| AI 预览 | AI 内容可生成预览，结果正确显示 |
| 模板保存 | 模板可保存到 OSS，元数据正确存储 |

### 6.2 非功能验收

| 指标 | 标准 |
|------|------|
| 页面加载时间 | < 3 秒 |
| API 响应时间 | < 500ms（不含外部接口） |
| AI 生成时间 | < 10 秒 |
| 浏览器兼容 | Chrome、Edge 最新版 |
